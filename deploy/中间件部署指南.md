## 1. 环境要求

| 工具 | 最低版本 | 说明 |
|------|----------|------|
| **Docker Desktop** | 4.x | 必须已启动（托盘图标变绿） |
| **Node.js** | 18.x+ | 推荐 20 LTS |
| **pnpm** | 8.x+ | `npm i -g pnpm` |
| **Git** | 2.x | - |

> ⚠️ **Windows 用户**：Docker Desktop 必须开启 WSL2 后端。如果遇到 `exec format error`，说明拉到了 arm64 镜像，docker-compose.yml 已配置 `platform: linux/amd64`。

## 2. 中间件部署（Docker）

所有中间件由 `D:\middleware\docker-compose.yml` 统一管理，分为两个 profile：

### 2.1 服务清单

#### 🟢 核心服务（core）— 项目运行必须

| 服务 | 镜像 | 端口 | 用途 |
|------|------|------|------|
| **MySQL** | mysql:8.0.36 | `3307`→3306 | 关系型数据库，存储用户、岗位、学习计划等结构化数据 |
| **Redis** | redis:7.2-alpine | 6379 | 缓存 & 会话，Token 存储、热点数据、分布式锁 |
| **MongoDB** | mongo:7 | 27017 | 文档数据库，对话历史、日志、JSON 文档 |
| **RabbitMQ** | rabbitmq:3.13-management | 5672 / 15672 | 消息队列，Agent 任务分发、异步事件 |

#### 🟡 按需服务（optional）— 特定功能需要时启动

| 服务 | 镜像 | 端口 | 用途 |
|------|------|------|------|
| **MinIO** | minio/minio | 9000 / 9001 | 对象存储，文件上传（图片、PDF、模型文件） |
| **Neo4j** | neo4j:5.22 | 7474 / 7687 | 图数据库，知识图谱、技能关系网络 |
| **Chroma** | chroma:0.4.20 | 8000 | 向量数据库，RAG 语义检索 |
| **SearXNG** | searxng/searxng | 8080 | 元搜索引擎，AI Agent 联网搜索 |
| **RedisInsight** | redis/redisinsight | 5540 | Redis 可视化管理工具 |

### 2.2 一键启动

```bash
# ⚡ 第一步：以管理员身份打开终端，停掉 Windows 自带 MySQL（避免端口冲突）
net stop MySQL
sc config MySQL start=demand

# ⚡ 第二步：启动核心服务（MySQL + Redis + MongoDB + RabbitMQ）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core up -d

# 可选：启动全部服务（核心 + MinIO + Neo4j + Chroma + SearXNG + RedisInsight）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core --profile optional up -d

# 可选：单独启动某个按需服务
docker compose -f D:\middleware\docker-compose.yml -p middleware up -d neo4j
```

### 2.3 验证中间件是否正常

```bash
# 查看所有容器状态（应该全部 Up）
docker ps

# 测试 MySQL 连接
docker exec mysql mysql -uroot -proot123 -e "SELECT 1"

# 测试 Redis 连接
docker exec redis redis-cli ping
# 预期输出: PONG

# 测试 MongoDB 连接
docker exec mongo mongosh -u root -p root --authenticationDatabase admin --eval "db.runCommand({ping:1})"
```

## 3. 后端启动（NestJS）

### 3.1 安装依赖

```bash
cd D:\X\ZhiPath\backend-ts
pnpm install
```

### 3.2 配置环境变量

```bash
# 复制示例配置
copy .env.example .env
```

然后编辑 `.env`，**必须修改**的配置项：

```ini
# ── MySQL ──（Docker MySQL 端口是 3307，不是 3306）
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3307
MYSQL_USER=root
MYSQL_PASSWORD=root123
MYSQL_DATABASE=zhipath

# ── MongoDB ──
MONGODB_URL=mongodb://root:root@127.0.0.1:27017/zhipath?authSource=admin
MONGODB_DATABASE=zhipath

# ── Redis ──
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# ── LLM（至少配一个） ──
# 选项 A：本地 Ollama（免费，需额外安装 https://ollama.com）
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:7b

# 选项 B：DeepSeek API（推荐，便宜好用）
DEEPSEEK_API_KEY=sk-xxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com

# 选项 C：OpenAI 兼容 API
OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_BASE_URL=https://api.openai.com/v1

# ── JWT ──
SECRET_KEY=your-secret-key-here
JWT_EXPIRES_IN=7d
```

> 💡 **可选配置**：Neo4j、MinIO、SearXNG、多模态（智谱/讯飞）等按需填写，不配不影响核心功能。

### 3.3 启动后端

```bash
# 开发模式（热更新）
pnpm run start:dev

# 生产模式
pnpm run build
pnpm run start:prod
```

后端默认运行在 `http://localhost:3000`，API 前缀 `/api`。

## 7. 常用命令

### Docker 管理

```bash
# 查看容器状态
docker ps -a

# 停止所有服务
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core --profile optional down

# 停止按需服务（保留核心）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile optional down

# 重启单个服务
docker compose -f D:\middleware\docker-compose.yml -p middleware restart redis

# 查看日志（实时跟踪）
docker logs -f mysql

# 查看最近 100 行日志
docker logs --tail 100 rabbitmq
```

### 数据备份

```bash
# 备份 MySQL
docker exec mysql mysqldump -uroot -proot123 --all-databases > D:\middleware\backup\mysql_$(date +%Y%m%d).sql

# 备份 MongoDB
docker exec mongo mongodump --username root --password root --authenticationDatabase admin --out /data/db/backup

# Redis RDB 快照已自动保存在 D:\middleware\redis-data\dump.rdb

# 通用：停止服务后直接复制 D:\middleware\mysql-data 等目录
```

## 8. 故障排查

| 问题 | 排查方法 |
|------|----------|
| **连不上 MySQL** | 确认连的是 `3307` 不是 `3306`，用 `docker ps` 确认容器 Running |
| **Navicat 连不上** | Host 用 `127.0.0.1`（不要带空格），Port 是 `3307` |
| **端口被占用** | `powershell Get-NetTCPConnection -LocalPort 3307` 查看占用进程 |
| **Docker 容器启动失败** | `docker logs <容器名>` 查看错误日志 |
| **exec format error** | arm64/amd64 不匹配，`docker pull --platform linux/amd64 <镜像>` 重新拉取 |
| **Chroma 启动卡死** | compose 已覆盖 entrypoint 跳过 pypi 安装，如手动启动需加 `--entrypoint ""` |
| **SearXNG 配置** | 编辑 `D:\searxng-docker\settings.yml`，重启容器生效 |
| **后端启动报数据库连接失败** | 确认 Docker MySQL 容器已启动，`.env` 中端口是 `3307` |
| **前端请求 404** | 确认后端已启动在 `localhost:3000`，Vite 代理配置正确 |
