## P1：跨路由拖拽 → "派遣模式"重构

### 问题根因

代码库实际结构：

```
App.tsx
  ├── <UserLayout />          ← 侧边栏导航 + <Outlet />
  │   ├── /user/chat          → <Chat />（内嵌 AgentPanel 侧边栏）
  │   ├── /user/agent-office  → <AgentOffice />（独立页面）
  │   ├── /user/profile       → <Profile />
  │   ├── /user/learning      → <LearningPaths />
  │   └── ...
```

**已有的拖拽**（`AgentPanel.tsx:116-120` → `Chat.tsx:509-523`）：
- AgentPanel 在 Chat 页面内部，用 `native HTML5 draggable` + `onDragStart`
- Chat 输入框用 `onDrop` 接收，通过 `application/zhipath-agent` MIME type 传递 agentKey
- **这在同一个 DOM 树内有效，已经可用 ✅**

**不可行的拖拽**：
- 从 `/user/agent-office` 拖到 `/user/chat` — 两个路由，组件不同树，拖拽跨路由会中断
- 从 Office 拖到 Profile 页的图谱 — 同理

```typescript
// ── 新增：Agent 派遣菜单组件 ──
// components/office/AgentDispatchMenu.tsx

interface DispatchMenuProps {
  agentKey: string;          // 'path' | 'exam' | 'video' | ...
  position: { x: number; y: number };
  onClose: () => void;
  contextNodeId?: string;    // 如果从图谱节点触发
  contextPathId?: number;    // 如果从路径卡片触发
}

export function AgentDispatchMenu({ agentKey, position, onClose, contextNodeId, contextPathId }: DispatchMenuProps) {
  const navigate = useNavigate();

  const dispatchToChat = () => {
    // 携带参数跳转到 Chat，Chat 页检测后自动发送
    navigate(`/user/chat?agent=${agentKey}&auto=1`);
    onClose();
  };

  const dispatchToNode = () => {
    // 跳转到 Profile 页的图谱，定位到目标节点
    navigate(`/user/profile?focusNode=${contextNodeId}&agent=${agentKey}`);
    onClose();
  };

  const dispatchToPath = () => {
    // 直接调用后端 API 绑定 Agent 到路径
    bindAgentToPath(agentKey, contextPathId!).then(() => {
      toast.success('Agent 已绑定到学习路径');
    });
    onClose();
  };

  return (
    <div className="dispatch-menu" style={{ left: position.x, top: position.y }}>
      <button onClick={dispatchToChat}>
        💬 派遣到对话 — {AGENT_DROP_PROMPTS[agentKey]}
      </button>
      {contextNodeId && (
        <button onClick={dispatchToNode}>
          🎯 针对此技能执行任务
        </button>
      )}
      {contextPathId && (
        <button onClick={dispatchToPath}>
          📚 绑定为路径导师
        </button>
      )}
    </div>
  );
}
```

```typescript
export default function Chat() {

    const agentKey = searchParams.get('agent');

    if (agentKey && auto === '1') {
      const prompt = AGENT_DROP_PROMPTS[agentKey] || `请帮我处理${agentKey}相关任务`;
      setInput(prompt);
      // 自动发送
      setTimeout(() => {
        handleSend(prompt);
        // 清除 URL 参数，避免刷新重复触发
        setSearchParams({});
      }, 300);
    }
  }, [searchParams]);
}
```

## P2：react-dnd × R3F → 原生 pointer 事件

### 问题根因

方案 Phase 4 引入 `@react-three/fiber`（3D 图谱），Phase 5 引入 `react-dnd`（拖拽）。
但：
- react-dnd 的 `HTML5Backend` 监听 DOM `mousedown/mousemove/mouseup` 事件
- R3F 的 `<Canvas>` 用 WebGL 渲染，内部对象没有 DOM 节点
- react-dnd 的 `useDrop` **无法检测到 Three.js mesh 上的 drop 事件**
- 即使改为 `@dnd-kit` 也有同样问题

### 修订方案：不引入 react-dnd，分场景用不同技术

```
┌─────────────────────────────────────────────────────────────────┐
│                    拖拽/交互技术矩阵                              │
│                                                                  │
│  场景                      技术选择              状态             │
│  ──────────────────────────────────────────────────────────────  │
│  AgentPanel → Chat 拖拽    原生 HTML5 DnD        ✅ 已有         │
│  Office 内部工位拖拽       原生 HTML5 DnD        ✅ 已有         │
│  Agent → 图谱节点          R3F pointer 事件      🆕 新建         │
│  Agent → 图谱节点          点击派遣菜单          🆕 新建（备选）  │
│  图谱内节点拖拽重排        R3F + d3-force        🆕 自动布局     │
│  Agent → 路径卡片          原生 HTML5 DnD        🆕 同页内有效   │
│                                                                  │
│  结论：不引入 react-dnd / @dnd-kit                                │
│  原因：已有的原生 DnD 够用，3D 场景用 R3F 原生事件               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 图谱节点接收 Agent 的方案

方案 A（推荐）：**派遣菜单方式**，无需拖拽到图谱

```
用户在 Office 点击 Agent → 派遣菜单 → "针对某技能执行任务"
  → 跳转到 Profile 页，图谱自动聚焦到目标节点
  → 节点旁弹出快捷菜单（生成路径/出题/生成视频）
  → 用户选择后执行
```

方案 B（高级，Phase 9+ 实现）：**同页内 AgentPanel 拖拽到图谱**

```typescript
// 仅在 Profile 页的图谱区域内生效
// Profile 页内嵌 AgentPanel（复用 Chat 页的同款组件）
// AgentPanel 的 native DnD → 图谱区域的 onDragOver/onDrop

// Profile.tsx
<div className="profile-layout">
  <AgentPanel isOpen={panelOpen} onToggle={togglePanel} />
  <div
    className="graph-area"
    onDragOver={(e) => e.preventDefault()}
    onDrop={(e) => {
      const agentKey = e.dataTransfer.getData('application/zhipath-agent');
      if (agentKey) {
        // 获取鼠标位置对应的最近节点
        const nearestNode = findNearestNode(e.clientX, e.clientY);
        if (nearestNode) {
          showNodeAgentMenu(agentKey, nearestNode.id);
        }
      }
    }}
  >
    <SkillGraphScene ... />
  </div>
</div>
```

**注意：** native HTML5 `onDrop` 是 DOM 事件，即使目标区域上面覆盖了 R3F Canvas，DOM 事件仍然能冒泡到外层 div。所以这个方案**可行**，但 drop 位置无法精确映射到 3D 节点，需要做"最近节点"计算。

#### 3.4 后端实现：绑定路径 Agent 主动推送

```typescript
// learning.service.ts — commitSkill 扩展（+20行）
async commitSkill(planId: string, skillName: string, message?: string) {
  // ... 现有逻辑 ...

  // 新增：检查是否有绑定的 Agent
  const plan = await this.getPlan(planId);
  if (plan.boundAgentType) {
    // 异步触发 Agent 辅导（不阻塞主流程）
    this.triggerBoundAgent辅导(plan, skillName, commit).catch(() => {});
  }

  return result;
}

private async triggerBoundAgent辅导(plan: any, skillName: string, commit: any) {
  // 1. 生成辅导内容（LLM）
  const prompt = `用户刚刚完成了「${skillName}」的学习。
    路径：${plan.name}，当前进度：${plan.progress}%。
    请给出简短的鼓励和下一步建议（50字以内）。`;
  const advice = await this.llmService.chat(prompt);

  // 2. 推送通知
  await this.notificationService.create({
    userId: plan.userId,
    title: `🤖 ${AGENT_LABELS[plan.boundAgentType]} 的建议`,
    content: advice,
    type: 'agent_advice',
    metadata: { planId: plan.id, agentType: plan.boundAgentType },
  });

  // 3. SSE 实时推送
  this.eventsService.emit(plan.userId, {
    type: 'agent_advice',
    data: {
      agentType: plan.boundAgentType,
      planId: plan.id,
      advice,
      skillName,
    },
  });
}
```

```typescript
// graph-enhanced.service.ts
async generateGraphSnapshot(userId: string): Promise<GraphSnapshot> {
  const userSkills = await this.getUserSkills(userId);

  // 1. 从 seed 构建基础图
  const seedNodes = SKILL_GRAPH_SEED.map(seed => {
    const userSkill = userSkills.find(s => s.skillName === seed.name);
    return {
      id: seed.name,
      name: seed.name,
      category: seed.category,
      cluster: seed.cluster,
      mastery: userSkill?.mastery || 0,
      trustWeight: userSkill?.trustWeight || 0,
      effectiveMastery: (userSkill?.mastery || 0) * (userSkill?.trustWeight || 0),
      lastUpdated: userSkill?.updatedAt || 0,
      prerequisites: seed.prerequisites,
      relatedSkills: seed.related,
      level: seed.level,
    };
  });

  // 2. 追加用户自定义技能（不在 seed 中的）
  const seedNames = new Set(SKILL_GRAPH_SEED.map(s => s.name));
  const customSkills = userSkills.filter(s => !seedNames.has(s.skillName));
  customSkills.forEach(skill => {
    seedNodes.push({
      id: skill.skillName,
      name: skill.skillName,
      category: skill.category || '其他',
      cluster: 'custom',
      mastery: skill.mastery,
      trustWeight: skill.trustWeight,
      effectiveMastery: skill.mastery * skill.trustWeight,
      lastUpdated: skill.updatedAt,
      prerequisites: [],
      relatedSkills: [],
      level: 3,
    });
  });

  // 3. 构建边
  const edges: SkillEdge[] = [];
  seedNodes.forEach(node => {
    node.prerequisites.forEach(prereqName => {
      const prereq = seedNodes.find(n => n.name === prereqName);
      if (prereq) {
        edges.push({ from: prereq.id, to: node.id, type: 'prerequisite', strength: 0.8 });
      }
    });
    node.relatedSkills.forEach(relatedName => {
      const related = seedNodes.find(n => n.name === relatedName);
      if (related && node.id < related.id) { // 避免重复边
        edges.push({ from: node.id, to: related.id, type: 'related', strength: 0.4 });
      }
    });
  });

  // 4. 过滤掉 mastery=0 的孤立节点（可选，减少渲染压力）
  const activeNodes = seedNodes.filter(n => n.mastery > 0 || edges.some(e => e.from === n.id || e.to === n.id));

  return { nodes: activeNodes, edges, metrics: this.calculateMetrics(activeNodes) };
}
```

## P5：GraphSnapshot 存储策略

### 问题

原方案每次 commit 存完整快照（nodes + edges JSON），50 个技能的用户单个 snapshot ~50KB，频繁 commit 导致存储膨胀。

### 修订方案：快照 + 增量 delta

```
存储策略：
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│  方式一：全量快照（原方案）                                       │
│  commit 1: [{React: 65}, {Hooks: 30}, ...]  50KB                │
│  commit 2: [{React: 70}, {Hooks: 40}, ...]  50KB                │
│  commit 3: [{React: 70}, {Hooks: 45}, ...]  50KB                │
│  总计: 150KB（100 commit = 5MB）                                  │
│                                                                  │
│  方式二：首全量 + 后增量（修订方案）                               │
│  commit 1: 全量快照                     50KB（基准）              │
│  commit 2: delta only {React +5, Hooks +10}  0.5KB              │
│  commit 3: delta only {Hooks +5}             0.2KB              │
│  ...每 20 个 commit 存一次全量快照（checkpoint）                  │
│  总计: ~52KB（100 commit ≈ 50KB + 2KB）                          │
│                                                                  │
│  回放时：找到最近的全量快照 + 后续 deltas 逐步 apply              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 数据库修订


### 回放逻辑

```typescript
// graph-enhanced.service.ts
async getSnapshotAt(userId: string, targetCommitId: string): Promise<GraphSnapshot> {
  // 1. 找到 target 之前最近的全量快照
  const baseSnapshot = await this.snapshotRepo.findOne({
    where: { userId, snapshotType: 'full', createdAt: LessThan(targetDate) },
    order: { createdAt: 'DESC' },
  });

  // 2. 获取 base 之后到 target 之间的所有 delta
  const deltas = await this.snapshotRepo.find({
    where: {
      userId,
      snapshotType: 'delta',
      createdAt: Between(baseSnapshot.createdAt, targetDate),
    },
    order: { createdAt: 'ASC' },
  });

  // 3. 逐步 apply delta 到 base
  let snapshot = baseSnapshot.toGraphSnapshot();
  for (const delta of deltas) {
    snapshot = applyDelta(snapshot, delta.toDelta());
  }

  return snapshot;
}
```

```typescript
// hooks/useForceLayout.ts
function useForceLayout(nodes: SkillGraphNode[], edges: SkillEdge[]) {
  const [positions, setPositions] = useState([]);
  const [settled, setSettled] = useState(false);

  useEffect(() => {
    let tickCount = 0;
    const MAX_TICKS = 300;

    const sim = forceSimulation(nodes)
      .force('charge', forceManyBody().strength(-200))
      .force('link', forceLink(edges).distance(2))
      .force('center', forceCenter())
      .alphaDecay(0.02);

    sim.on('tick', () => {
      tickCount++;
      if (tickCount >= MAX_TICKS) {
        sim.stop();
        setSettled(true);
      }
      setPositions(sim.nodes().map(n => [n.x, n.y, n.z]));
    });

    return () => sim.stop();
  }, [nodes.length, edges.length]);  // 只在拓扑变化时重新计算

  return { positions, settled };
}
```

## P7：Toast 通知防抖

### 问题

方案中多个事件触发 toast：`path_generated`、`skill_completed`、`match_updated`、`agent_task_completed`、`agent_task_started`、`resource_ready`。连续操作时会轰炸用户。

### 修订方案

```typescript
// hooks/useWorkspaceSync.ts — toast 防抖
import { useRef } from 'react';

// Toast 合并队列
const pendingToasts = new Map<string, { count: number; timer: ReturnType<typeof setTimeout> }>();

function debouncedToast(key: string, message: string, icon?: string) {
  const existing = pendingToasts.get(key);

  if (existing) {
    existing.count++;
    clearTimeout(existing.timer);
    existing.timer = setTimeout(() => {
      const count = existing.count;
      toast.success(count > 1 ? `${message}（共 ${count} 项）` : message, { icon });
      pendingToasts.delete(key);
    }, 2000);
  } else {
    const entry = { count: 1, timer: setTimeout(() => {
      toast.success(message, { icon });
      pendingToasts.delete(key);
    }, 2000) };
    pendingToasts.set(key, entry);
  }
}

// 使用方式（替代原方案中的直接 toast 调用）
case 'skill_completed':
  debouncedToast('skill', `🎉 技能「${e.skillName}」学习完成！`, '🎉');
  break;

case 'match_updated':
  // 匹配度变化不防抖，每次都显示（重要信息）
  toast(`🎯 匹配度更新为 ${e.newScore}%`, {
    icon: e.newScore >= 80 ? '🚀' : '📈',
  });
  break;
```

### Toast 优先级分级

| 优先级 | 事件 | 行为 |
|--------|------|------|
| **P0 立即显示** | 匹配度 ≥ 80%、考试结果 | 直接 toast，不防抖 |
| **P1 短防抖 2s** | skill_completed、agent_task_completed | 合并同类，显示计数 |
| **P2 长防抖 5s** | resource_ready、today_tasks_refresh | 静默或合并 |
| **P3 不显示** | agent_task_started、agent_status | 仅更新 UI 状态，不弹 toast |

```typescript
const AGENT_ONBOARDING_KEY = 'zhipath_agent_onboarding_done';

export function useAgentOnboarding() {
  const [showGuide, setShowGuide] = useState(false);

  useEffect(() => {
    const done = localStorage.getItem(AGENT_ONBOARDING_KEY);
    if (!done) {
      // 延迟 2 秒后显示引导（让页面先加载完）

      return () => clearTimeout(timer);
    }
  }, []);

  const dismiss = () => {
    localStorage.setItem(AGENT_ONBOARDING_KEY, '1');
    setShowGuide(false);
  };

  return { showGuide, dismiss };
}
```
