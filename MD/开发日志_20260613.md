### 4. Vite HMR 修复
- **问题**：WSL2 + `/mnt/d/`（NTFS 挂载盘）下 inotify 无法监听文件变化，HMR 失效
- **修复**：`vite.config.ts` 添加 `server.watch.usePolling: true, interval: 300`

### 5. AI 对话生成路径卡住问题修复
- **问题**：用户在 Chat 中说"制定学习计划"，AI 回复"学习路径生成中..."后永远卡住
- **根因**：`ActionExecutorService.generatePath()` 把任务塞进原始 Redis（`redis.setex` + `redis.lpush`），但 BullMQ worker 只消费 BullMQ 队列，两边不搭，任务永远无人处理
- **修复**：改为直接调用 `PlannerAgentService.generatePath()` 同步生成，和 `POST /api/user/plans` 走同一条链路
- **文件**：
  - `chat.module.ts`：导入 `PlannerModule`
  - `action-executor.service.ts`：注入 `PlannerAgentService`，重写 `generatePath` 方法
  - `chat.service.ts`：`aiSummarize` 新增 `path_generated` 类型处理

## 智能体系统审计结果

### 架构问题

**两套 LLM 客户端并行运行：**
- `LlmService`（NestJS，openai SDK）→ 聊天、意图识别、出题
- `shared.ts`（原始 fetch）→ 5 个内容 Agent（讲义/阅读/代码/路径/评估）

### 评分总览

| 智能体 | 评分 | 核心问题 |
|--------|------|----------|
| PlannerAgentService（路径生成） | ✅ 可靠 | 纯算法，模板只覆盖8个方向 |
| MatchAgentService（岗位匹配） | ✅ 可靠 | 纯算法，项目匹配太粗糙 |
| TutorPromptService（提示词） | ✅ 优秀 | 全库最好的提示词设计 |
| IntentRouterService（意图识别） | ⚠️ 近可用 | 关键词顺序敏感 |
| AgentEngineService（聊天兜底） | ⚠️ 近可用 | 正则提取action块脆弱 |
| ActionExecutorService（动作执行） | ⚠️ 中风险 | 正则极脆弱，资源库硬编码 |
| 5个内容Agent | ⚠️ 中风险 | 独立fetch路径，无max_tokens |
| LlmService（LLM基础设施） | ❌ 需修复 | 无重试/超时/降级/成本控制 |

### 待修复优先级

| **P0** | LlmService 加重试+超时+降级 |
| **P0** | 统一两套LLM客户端到 LlmService |

| **P1** | ActionExecutor正则改为宽松匹配 |
