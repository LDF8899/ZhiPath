## 三页联动 × 3D 知识图谱 全量实施

### 目标

基于 `三页联动×3D知识图谱融合方案 v3.0` 和 `三页联动方案修订补丁_v3.1.md`，实施四大核心能力：
1. 跨页面事件总线（Chat ↔ Learning ↔ Office ↔ Profile 数据互通）
2. 3D 知识图谱可视化（替换平面雷达图）
3. 智能体办公室改造（1053行单文件拆分 + 点击派遣机制）
4. 后端 SSE 事件桥接（Notification → SSE → 前端实时响应）

### 问题诊断

| 问题 | 严重度 | 描述 |
|------|--------|------|
| 三页断裂 | HIGH | Chat 生成路径 → Learning 不知道；Learning 完成技能 → Chat 无反馈 |
| AgentOffice 单文件 | HIGH | 1053 行 monolith，本地 state，无事件发射 |
| 无能力可视化 | HIGH | 学习数据只有进度条，无技能关系/成长动画 |
| SSE 和 Notification 断开 | MEDIUM | NotificationService 有 5 个便利方法无人调用 |
| Agent 配置双轨制 | MEDIUM | AGENT_CONFIGS(intent-based) vs AGENT_LABELS(backend-based) 互不认识 |
| 拖拽跨路由不可行 | HIGH | react-dnd HTML5Backend 无法跨 SPA 路由（v3.1 修订发现） |
| react-dnd × R3F 不兼容 | HIGH | HTML5Backend 无法检测 WebGL Canvas 内 drop（v3.1 修订发现） |

### 修订策略（v3.1 → v3.0 落地）

在实施前，先对原始方案做了 8 项修订（见 `三页联动方案修订补丁_v3.1.md`），最关键的三项：

1. **跨路由拖拽** → 改为"点击派遣菜单"模式（`navigate('/user/chat?agent=path&auto=1')`）
2. **react-dnd** → 不引入，保持已有原生 HTML5 DnD + 新增 pointer 事件方案
3. **Agent 绑定路径** → 补完整闭环（数据库字段 + 触发规则 + 推送渠道）

### 核心联动效果（修复后）

```
三页事件管道：
  Chat 生成路径 ──emit path_generated──► LearningPaths 自动刷新
  KnowledgeDetail 完成技能 ──emit skill_completed──► 图谱变形 + toast + 匹配度更新
  考试通过 ──emit exam_completed──► 信任权重提升 + 图谱透明度变化
  AgentOffice 任务完成 ──SSE──► useWorkspaceSync → toast(防抖2s)
  Agent 完成学习路径生成 ──SSE──► 图谱动画 + LearningPaths 刷新

点击派遣管道：
  Office 点击 Agent → AgentDispatchMenu → navigate ?agent=path&auto=1
  → Chat 自动填充 prompt → handleSend → 后端处理 → SSE 结果
  → emit 事件 → 全局 toast + 图谱更新

Agent 绑定路径管道（P3）：
  路径卡片拖拽/点击 → bindAgentToPath API → learning_plans.bound_agent_type
  → commitSkill 时检测绑定 → 触发 LLM 辅导 → NotificationService.create()
  → SSE emit agent_advice → 前端 toast("💬 Agent建议: ...")

3D 图谱管道：
  Profile 页 → SkillGraphContainer → WebGL 检测
    有 WebGL → SkillGraphScene (R3F Canvas)
    无 WebGL → RadarChart 降级（复用现有组件）
  力导向布局 → 300 tick 上限 → 节点/边/聚类渲染
  delta 事件 → useGraphAnimation → 节点脉冲 + 边变色 + 新节点弹入
```
