## 二、后端差距

### 2.1 致命问题 (P0)

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| B1 | **`@CurrentUser('id')` 返回 undefined** | `chat.controller.ts`×4, `github.controller.ts`×1, `progress.controller.ts`×4 | JWT payload 是 `{sub, username, role}`，但代码读 `user.id` → 接口 500 |
| B2 | **ResourceProcessor 是空壳** | `queue/resource.processor.ts` | BullMQ 消费端只返回 `{status:'completed'}`，不调用实际生成逻辑 → 资源生成任务"假完成" |

### 2.2 功能缺失 (P1)

| # | 功能 | 宪法章节 | 当前状态 | 需要做的事 |
|---|------|----------|----------|------------|
| B3 | **JDParserAgent** | §2.2 | 完全不存在 | 新建 service，解析 JD 提取技能列表，写入 `job_positions_v3.required_skills` + 更新 Neo4j |
| B4 | **GitHub API 集成** | §16 | Controller 有 TODO，返回占位符 | 实现 repo 元数据采集、关键文件读取、目录树分析，接入 ProfileAgent |
| B5 | **SearXNG 资讯抓取** | §3.3, 附录D.4 | Controller 有 TODO，返回占位符 | 实现 SearXNG 客户端，定时抓取行业资讯，存入 `news_v3` |
| B6 | **邮件通知 (Nodemailer)** | §18.2 | 完全不存在 | 安装 nodemailer，实现邮件服务（投递结果通知、7天未登录召回、学习周报） |
| B7 | **匹配度权重与宪法不一致** | §9.1 | 代码：必须35%/加分15%/项目20%/考试15%/进度15% | 宪法要求校招：必须30%/加分15%/项目15%/考试20%/进度10%/速度10% |

### 2.3 功能不完整 (P2)

| # | 功能 | 当前状态 | 缺失部分 |
|---|------|----------|----------|
| B8 | 接口返回 snake_case | 部分接口可能返回 camelCase | 需全面检查所有 Controller 序列化输出 |
| B9 | MongoDB 对话历史冷归档 | Redis 热数据存在 | 缺少 90 天后归档到 MongoDB 的定时任务 |
| B10 | 通知 90 天过期清理 | `notifications_v3` 有数据 | 缺少定时清理过期通知的逻辑 |
| B11 | 知识库每周定期更新 | 知识库 CRUD 存在 | 缺少 SearXNG 搜索→对比→标记审查 的自动流程 |
| B12 | 新岗位录入后联动 | Neo4j 更新存在 | 缺少"触发所有用户匹配度重算 + 匹配度≥60%通知"的完整链路 |

## 三、前端差距

### 3.1 组件缺失 (P0)

| # | 组件 | 宪法章节 | 影响范围 |
|---|------|----------|----------|
| F1 | **日历组件** | §6.2 | 首页无日历展示学习情况 |
| F2 | **Monaco Editor** | §7.1 | 编程题用 `<textarea>` 代替，无语法高亮/自动补全 |

| F4 | **拖拽排序** | §11.2 | 智能体办公室无拖拽功能，未安装 dnd 库 |

### 3.2 页面功能不完整 (P1)

| # | 页面 | 缺失功能 | 宪法要求 |
|---|------|----------|----------|
| F5 | **首页 Dashboard** | 缺日历、匹配度分析、推荐岗位、快捷操作、智能体小部件 | §6.1 完整布局 |
| F6 | **学习路径** | 无"岗位驱动/自选学习"双 Tab 切换 | §4.1 多计划架构 |
| F7 | **学习详情** | 路由是 `/user/knowledge/:skill` 非 `/user/learning/:id`；缺多模态资源（视频/编程题/图谱） | §7.1 多模态资源 |
| F8 | **岗位详情** | 缺"加入学习计划"按钮和"生成简历"按钮 | §10.1 岗位与计划联动 |
| F9 | **学习记录** | 用列表展示，无 Git commit 风格的可视化（提交点/分支/对比图） | §13.1 Git Commit 模型 |
| F10 | **简历管理** | API 和后端 PDF 接口都在，但页面没有"导出 PDF"按钮 | §12.2 简历生成流程 |
| F11 | **智能体办公室** | 有任务列表，但缺拖拽排序优先级 | §11.2 用户交互 |
| F12 | **Onboarding** | 只有静态表单，缺 AI 对话模式 | §5.3 两种模式 |

### 3.3 AI 浮窗 (P1)

| # | 问题 | 宪法要求 |
|---|------|----------|
| F13 | **上下文感知缺失** | §17.1 要求不同页面有不同 AI 角色和 session 隔离，但当前浮窗不读路由，所有页面行为相同 |
| F14 | **三层记忆架构未实现** | §17.2 要求 Global Profile / Page Session / Temporary 三层，当前只有全局对话 |
| F15 | **资讯页临时 session** | §17.3 要求对话不存 MongoDB、不影响画像，当前未做隔离 |

### 3.4 技术规范 (P2)

| # | 问题 | 宪法要求 |
|---|------|----------|
| F16 | **前端类型命名不一致** | §20.1 要求前端类型字段与后端 JSON 一致（snake_case），但除 `DashboardData` 外全部用 camelCase |
| F17 | **Login/Register 死代码** | 路由重定向到 Landing 页，独立组件未被使用 |
