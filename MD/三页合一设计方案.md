# 三页合一设计方案：Chat × Learning × Agent Office 统一工作空间

> 智途 ZhiPath 前端整合设计
> 版本：v1.0 | 日期：2026-06-19
> 状态：**待审核**

---

## 一、现状诊断

### 1.1 三个页面现状

| 页面 | 路由 | 文件 | 状态管理 | 侧边栏 |
|------|------|------|----------|---------|
| 对话 | `/user/chat` | `pages/user/Chat.tsx` (630行) | `useChatStore` (sessionStorage) + `useOfficeStore` (localStorage) | 左：会话列表，右：AgentPanel |
| 学习路径 | `/user/learning` | `pages/user/LearningPaths.tsx` | local useState（无 store） | 无 |
| 智能体办公室 | `/user/agent-office` | `pages/user/AgentOffice.tsx` (1053行) | local useState + 8秒轮询 | 无（内嵌完整办公室） |

### 1.2 三个核心断裂

```
断裂 1：布局断裂
┌─────────────────────────────────────────────────────┐
│  Chat 有自己的侧边栏 ─┐                               │
│  Learning 没有侧边栏 ─┤ 三个页面各自为营，导航不统一     │
│  AgentOffice 自己布局 ─┘                               │
└─────────────────────────────────────────────────────┘

断裂 2：状态断裂
┌─────────────────────────────────────────────────────┐
│  Chat: useChatStore (sessionStorage)                 │
│  Learning: useState (内存，页面切换即丢失)             │
│  AgentOffice: useState + 8秒轮询 (独立数据源)          │
│                                                      │
│  三者互不通信，无共享状态                               │
└─────────────────────────────────────────────────────┘

断裂 3：交互断裂
┌─────────────────────────────────────────────────────┐
│  Chat 生成路径 → Learning 不知道 → 用户手动刷新        │
│  Learning 完成技能 → Chat 不知道 → 无鼓励反馈          │
│  AgentOffice 任务完成 → 仅轮询可见 → Chat 无感知        │
└─────────────────────────────────────────────────────┘
```

### 1.3 Agent 模型冲突

| 维度 | Chat 侧 (AGENT_CONFIGS) | AgentOffice 侧 (AgentProfile) |
|------|--------------------------|-------------------------------|
| 数据源 | 前端硬编码常量 | 后端数据库 (agent_profiles_v3) |
| agent key | `generate_video` | `video` (agent_type) |
| agent id | `video` | 数据库自增 id |
| 头像 | 固定 12 种动物 | 用户自选（12种动物 + 自定义颜色） |
| 状态 | Zustand store，SSE 驱动 | 后端枚举，轮询驱动 |
| 数量 | 固定 11 个（不含 chat） | 用户按需招聘，数量不定 |

### Phase 2：统一状态（数据层）

> 目标：三端共享关键数据，消除重复 API 调用

#### 2.1 新建 workspace.ts store

位置：`frontend/src/stores/workspace.ts`

```ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface WorkspaceState {
  // ── 全局用户数据 ──
  matchScore: number;                    // 匹配度（TopBar 显示）
  currentPlanId: number | null;          // 当前活跃学习计划 ID
  dailyHours: number;                    // 每日学习时长
  targetJobTitle: string;                // 目标岗位名称

  // ── Agent 状态（合并自 office.ts）──
  activeAgent: string | null;
  agentStatuses: Record<string, 'idle' | 'working' | 'completed'>;
  agentBusyCount: number;               // Rail 底部指示灯
  agentIdleCount: number;

  // ── 任务队列 ──
  tasks: TaskProgress[];                 // 前端 SSE 任务

  // ── 跨页面事件 ──
  lastEvent: WorkspaceEvent | null;      // 最近一次跨页面事件

  // ── Actions ──
  setMatchScore: (score: number) => void;
  setCurrentPlan: (id: number) => void;
  setActiveAgent: (agent: string | null) => void;
  setAgentStatus: (agent: string, status: 'idle' | 'working' | 'completed') => void;
  addTask: (task: TaskProgress) => void;
  updateTask: (taskId: string, patch: Partial<TaskProgress>) => void;
  emit: (event: WorkspaceEvent) => void; // 发射跨页面事件
}

// 事件类型
type WorkspaceEvent =
  | { type: 'path_generated'; planId: number; planName: string }
  | { type: 'skill_completed'; skillName: string }
  | { type: 'exam_generated'; examId: number; skill: string }
  | { type: 'video_ready'; taskId: string; skillName: string }
  | { type: 'match_updated'; newScore: number }
  | { type: 'task_completed'; taskId: string; agentType: string };
```

#### 2.2 保留 useChatStore

`useChatStore` 保持不变（消息数据仍用 sessionStorage），但增加事件发射：

```ts
// Chat.tsx handleSend 中，收到 agent 响应后：
```

`useOfficeStore` 保持向后兼容，但逐步将数据迁移至 workspace store。过渡期内两个 store 并存，通过 subscribe 桥接。

### Phase 3：双向同步（交互层）

> 目标：Chat 操作 ↔ Learning 状态 实时联动

#### 3.1 新建 useWorkspaceSync.ts

位置：`frontend/src/hooks/useWorkspaceSync.ts`

在 WorkspaceLayout 中调用，监听 workspace store 事件并执行副作用：

```ts
export function useWorkspaceSync() {
  const navigate = useNavigate();
  const showToast = useToast();

  useEffect(() => {
    return useWorkspaceStore.subscribe(
      (state) => state.lastEvent,
      (event) => {
        if (!event) return;

        switch (event.type) {
          case 'path_generated':
            showToast('学习路径已生成！点击查看', 'success');
            // 可选：自动刷新 Learning 页面数据
            break;

          case 'skill_completed':
            showToast(`🎉 恭喜完成「${event.skillName}」！`, 'success');
            // 更新匹配度
            workspaceStore.getState().refreshMatchScore();
            break;

          case 'exam_generated':
            showToast('练习题已生成', 'success');
            break;

          case 'match_updated':
            if (event.newScore >= 80) {
              showToast(`🎯 匹配度达到 ${event.newScore}%！可以投递了`, 'success');
            }
            break;

          case 'video_ready':
            showToast('教学视频已就绪', 'success');
            break;
        }
      }
    );
  }, []);
}
```

#### 3.2 Chat → Learning 流程

```
用户在 Chat 说 "帮我生成学习路径"
    ↓
Chat.tsx handleSend() → POST /api/user/chat
    ↓
后端 LangGraph → generatePathNode → 返回 { actions: [{ type: 'path_generated', data: {...} }] }
    ↓
Chat.tsx 收到响应 → workspaceStore.emit({ type: 'path_generated', planId, planName })
    ↓
useWorkspaceSync 捕获事件 → showToast("学习路径已生成")
    ↓
用户切到 /user/learning → paths 已从 workspaceStore 缓存读取 → 立即显示
```

#### 3.3 Learning → Chat 流程

```
用户在 Learning 完成一个技能
    ↓
LearningPaths.tsx → markSkillComplete() → workspaceStore.emit({ type: 'skill_completed', skillName })
    ↓
useWorkspaceSync 捕获 → showToast("🎉 恭喜完成...")
    ↓
workspaceStore.matchScore 更新（后端返回新匹配度）
    ↓
TopBar 匹配度 pill 实时更新
```

#### 3.4 Agent Office → Chat 流程

```
用户在 AgentOffice 派发任务
    ↓
createAgentOfficeTask() → 后端创建 AgentTask → agent 变为 busy
    ↓
workspaceStore.setAgentStatus(agentType, 'working')
    ↓
Rail 底部指示灯：⚡ 3忙（实时更新）
    ↓
AgentPanel（Chat 页可见）显示该 agent 工作中
    ↓
后端任务完成 → SSE event → workspaceStore 更新 → AgentPanel 显示结果
```

## 七、风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| Chat.tsx 改动较大（去顶层 layout） | 可能引入 UI 回归 | 逐步迁移，保留旧 layout 作为 fallback |
| workspace store 与 office store 并存期 | 数据不一致 | subscribe 桥接，逐步迁移 |
| AgentPanel 从 Chat 提取后 props 传递复杂 | 代码可读性下降 | 用 Zustand store 替代 props drilling |
| 首屏加载变大（WorkspaceLayout 包裹所有页面） | 性能 | React.lazy 子路由 + WorkspaceLayout 保持轻量 |
| 旧路由 bookmark 失效 | 用户体验 | App.tsx redirect 旧路由到新路由 |

## 八、验收标准

1. **布局一致性**：在 `/user/chat`、`/user/learning`、`/user/agent-office` 间切换，左侧 Rail 和顶部 TopBar 不闪烁、不重载
2. **状态持久性**：在 Chat 生成学习路径后，不刷新页面切到 Learning，路径已显示
3. **事件联动**：在 Learning 完成技能 → TopBar 匹配度更新 + toast 提示
4. **Agent 可见性**：在 AgentOffice 派发任务 → Chat 页 AgentPanel 实时显示进度
5. **无回归**：原有 Chat 对话、Learning 路径浏览、AgentOffice 任务管理功能全部正常
