# 中间件部署指南

## 概述

本机有两个 MySQL 实例：

| 实例 | 版本 | 端口 | 用途 |
|------|------|------|------|
| Windows MySQL | 8.0.33 | 3306 | 本机旧服务（建议停用） |
| Docker MySQL | 8.0.36 | 3307 | 项目数据库 |

**当前选择：停掉 Windows MySQL 权限不够，所以把 Docker MySQL 改到 3307，避开冲突。**

## Docker 中间件清单

所有服务统一由 `D:\middleware\docker-compose.yml` 管理，分为两个 profile。这些中间件供本机多个项目共用，不限于某个单一项目。

| 服务 | 镜像 | 端口 | 用途 | 数据目录 |

| MySQL | mysql:8.0.36 | 3307→3306 | 关系型数据库，存储结构化业务数据（用户、订单、配置等） | `D:\middleware\mysql-data` |

## 一键启动

> **前提：Docker Desktop 必须已启动。** 如果没启动，先打开 `C:\Program Files\Docker\Docker\Docker Desktop.exe`，等托盘图标变绿后再执行以下命令。

以**管理员身份**运行终端：

```bash
# 先停掉 Windows MySQL（避免端口冲突）
net stop MySQL
sc config MySQL start=demand

# 启动核心服务（MySQL + Redis + MongoDB + RabbitMQ）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core up -d

# 启动全部服务（核心 + MinIO + RedisInsight + Neo4j + Chroma + SearXNG）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core --profile optional up -d

# 单独启动某个服务
docker compose -f D:\middleware\docker-compose.yml -p middleware up -d neo4j
```

## 停止 / 重启 / 日志

```bash
# 停止所有服务
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile core --profile optional down

# 停止按需服务（保留核心）
docker compose -f D:\middleware\docker-compose.yml -p middleware --profile optional down

# 重启单个服务
docker compose -f D:\middleware\docker-compose.yml -p middleware restart redis

# 查看所有容器状态
docker ps -a

# 查看某个服务日志（实时跟踪）
docker logs -f mysql

# 查看最近 100 行日志
docker logs --tail 100 rabbitmq
```

## 数据备份与恢复

所有数据通过 volume 挂载到 `D:\middleware\` 下的本地目录，备份即复制目录，恢复即放回原位。

```bash
# 备份 MySQL（用 mysqldump 导出 SQL）
docker exec mysql mysqldump -uroot -proot123 --all-databases > D:\middleware\backup\mysql_all_$(date +%Y%m%d).sql

# 备份 MongoDB
docker exec mongo mongodump --username root --password root --authenticationDatabase admin --out /data/db/backup

# 备份 Redis（RDB 快照已在 redis-data 目录中）
# 直接复制 D:\middleware\redis-data\dump.rdb

# 备份 Neo4j
docker exec neo4j neo4j-admin database dump neo4j --to-path=/data/backup

# 通用：直接打包整个数据目录（适用于所有服务）
# 停止服务后复制 D:\middleware\mysql-data、mongo-data 等目录即可
```

如果你之前有其他本地项目用的 Docker MySQL（端口 3306），用这个提示词让 Claude 帮你改：

> 项目数据库之前连的 Docker MySQL on port 3306，现在 Docker MySQL 移到 3307 了（Windows 自带 MySQL 占着 3306）。帮我找到项目里所有 MySQL 连接配置，把 port 从 3306 改成 3307，其他不变。

## 排查备忘

- **连不上数据库？** 先检查是不是连到 Windows MySQL（3306）了，应该连 3307
- **Navicat 连不上？** 确认 Host 是 `127.0.0.1`（IP 格式，不要带空格），Port 是 `3307`
- **端口冲突？** 用 `powershell Get-NetTCPConnection -LocalPort 3306` 看谁占了端口
- **Docker 连不上？** 用 `docker ps` 确认容器 Running，再用 `docker logs <name>` 看日志
- **容器平台警告 (arm64/amd64)？** 用 `docker pull --platform linux/amd64 <镜像>` 拉取正确架构的镜像后重建容器
- **Chroma 启动卡死 (pypi 超时)？** 默认入口脚本每次启动都重装 hnswlib，compose 已覆盖 entrypoint 跳过此步骤。如果手动运行 chroma 容器，需要加 `--entrypoint ""` 并指定 command
- **SearXNG 配置？** 配置文件在 `D:\searxng-docker\settings.yml`，修改后重启容器生效
