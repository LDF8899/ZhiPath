## 岗位部分联动项目（切片 6）

### 目标
岗位部分"几乎是假的" — 匹配度是简单技能命中率而非5因子算法，前端类型残缺，缺少筛选功能，企业信息空壳。让岗位真正联动项目其他模块。

### 问题诊断

| 问题 | 严重度 | 描述 |
|------|--------|------|
| 匹配度算法假的 | HIGH | Jobs列表和Dashboard用简单技能命中率（hit/total），JobDetail用真实5因子算法，三处不一致 |
| 前端Job类型残缺 | HIGH | 缺少 `level`, `deliveryThreshold`, `source`, `enterpriseId`；`requiredSkills` 类型是 `string[]` 而非 `Array<{name, weight}>` |
| 后端返回字段不全 | MEDIUM | `getJobs()` 和 `getJob()` 没返回 `level`, `deliveryThreshold` 等字段 |
| 筛选排序功能弱 | MEDIUM | 没有按级别筛选，薪资排序不工作 |
| 企业信息空壳 | LOW | JobDetail的企业信息只显示company字符串，没有关联enterprises_v3表 |

### 核心联动效果

```
匹配度一致性：
  Jobs列表 ──┐
  Dashboard ─┼── 全部使用 MatchAgentService.calculateMatch() 5因子算法
  JobDetail ─┘

岗位 ↔ 企业：岗位关联 enterprises_v3 表，展示企业名称 + 行业
岗位 ↔ 技能：skills 类型对齐后端 {name, weight} 格式
岗位 ↔ 筛选：支持按级别（junior/mid/senior）筛选
```

### 数据流（修复后）

```
GET /api/user/jobs?level=mid
  → JobsService.getJobs()
    → 查询 job_positions_v3（带 level 筛选）
    → MatchAgentService.calculateForAllJobs(userId)
      → 5因子加权：必须技能(35%) + 加分技能(15%) + 项目(20%) + 考试(15%) + 学习进度(15%)
    → 关联 enterprises_v3 查询企业信息
    → 按匹配度降序排序返回

GET /api/user/dashboard
  → DashboardService.getDashboard()
    → targetJob 匹配度改用 matchAgent.calculateMatch()
    → 返回 level, deliveryThreshold, source 等完整字段
```
