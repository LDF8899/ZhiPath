# 学习深度改进方案

> 日期：2026-06-14
> 问题：智能体输出的学习资料太肤浅，用户在学习页面学不到东西

---

## 一、问题诊断

### 1.1 讲义 = 百科词条，不是教学

**现状**：`lecture-agent.service.ts:139-166` 的 prompt 生成固定模板：概述→前置知识→核心概念→代码→实战→误区→总结。这是参考文档结构，不是教学结构。

**缺失**：
- 没有"为什么需要它"的动机铺垫
- 没有脚手架递进（先直觉→再定义→最后细节）
- 没有认知冲突设计（反直觉问题引发思考）
- 类比是碎片化的，不是贯穿全文的
- 没有"思考暂停"——learner 被动滚动到底

### 1.2 代码案例 = 展品，不是练习

**现状**：`code-agent.service.ts:67-85` 生成 3 个 15-30 行独立代码片段。用户看到完成品，前端"试一试"只是复制→运行→看结果。

**缺失**：
- 没有填空式练习（给 80% 代码，填关键 20%）
- 没有 Debug 练习（找错）
- 案例之间没有渐进关系
- 没有"没有 X 之前 vs 有了 X 之后"的对比

### 1.3 拓展阅读 = 目录，不是阅读

**现状**：`reading-agent.service.ts:60-77` 生成标题 + 100-200 字摘要 + 关键词标签。用户得到书单，不是阅读材料。

### 1.4 测验 = 记忆检测，不是理解检测

**现状**：练习题嵌在讲义 `---EXERCISES---` 块里，以选择题为主。

**缺失**：
- 没有概念反转题（常见误解判断）
- 没有迁移题（A 场景知识用到 B 场景）
- 没有解释题（用自己的话讲给别人听）
- 没有间隔重复

### 1.5 评估 = 假分数

**现状**：`assess-agent.service.ts` 的输入只是"是否点了标记已读""是否提交了测验"。LLM 基于极弱信号编造 0-100 分。

### 1.6 所有人看到相同内容

**现状**：`knowledge-base.service.ts` 把生成结果缓存为 `shared: true`，同一技能所有用户拿到完全一样的内容。

---

## 二、改进方案

### P0：Prompt 重写（改 3 个文件，效果最大）

#### 2.1 LectureAgent — 教学式讲义

改用 Socratic + 脚手架结构：
- 开篇用真实痛点/故事引入
- 先回答"为什么需要它"
- 贯穿全文的类比
- 关键节点设置"思考暂停"
- 代码展示"before vs after"对比
- 结尾开放思考题

#### 2.2 CodeAgent — 渐进式练习

改为 3 个渐进练习：
- setup（80% 代码）+ task（20% 填空）+ hint + solution + expected_output
- 练习 1 基础 → 练习 2 变体 → 练习 3 综合

#### 2.3 ReadingAgent — 深度短文

从"摘要目录"改为 2 篇 800-1200 字深度短文：
- "为什么"类：解释设计决策/技术选型
- "实战"类：mini-project walkthrough

### P1：前端交互增强

#### 2.4 代码填空模式

KnowledgeDetail.tsx 的代码 Tab 改为：
- 渲染 setup 代码到 CodeMirror
- task 部分用空白高亮
- 用户填写后对比 solution

#### 2.5 讲义分段 + 检查点

讲义切成 3-4 个 section，每个 section 结尾有 1 道理解检查题，通过后看下一个 section。

#### 2.6 "教别人"模式

讲义末尾加输入框，用户用自己的话解释概念，提交后 LLM 给反馈。

### P2：Quiz 增强

#### 2.7 题型多样化

在讲义 exercises 中增加：
- 概念反转题（判断常见误解 + 解释）
- 迁移题（学了 A 场景，应用到 B 场景）

### P3：间隔重复 + 个性化

#### 2.8 间隔重复队列

错过的题在 1 天/3 天/7 天后重新出现。

#### 2.9 去掉共享缓存

按用户 level 分缓存，因材施教。
