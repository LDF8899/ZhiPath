# 三页联动方案：Chat × Learning × AgentOffice 跨页面互通

> 智途 ZhiPath 前后端优化
> 版本：v1.0 | 日期：2026-06-19
> 原则：**每个页面保持自己的布局不变**，只加事件总线让三页互相感知

---

## 一、现状问题

### 1.1 三页断裂点

```
Chat 生成学习路径 → Learning 不知道 → 用户手动刷新
Learning 完成技能 → Chat 不知道 → 无鼓励反馈、匹配度不更新
AgentOffice 任务完成 → 仅页面内可见 → Chat/Learning 无感知
```

### 1.2 已有基础设施（可复用）

| 组件 | 状态 | 复用方式 |
|------|------|----------|
| `useSSE` hook | ✅ 已有，AgentPanel 和 Dashboard 在用 | 扩展到 Learning 页 |
| `EventsService` | ✅ 已有，内存 pub/sub | 补充缺失的 emit 调用 |
| `NotificationService` | ✅ 已有，5 个便利方法无人调用 | 在关键节点触发 |
| `useOfficeStore` | ✅ 已有，Chat+AgentPanel 共用 | 补充 workspace 共享字段 |
| `useChatStore` | ✅ 已有 | 不改结构，加事件发射 |

### 1.3 核心差距

- **后端**：AgentOffice 的 `executeTask` 不发 SSE 事件（queue processor 发，但 office 直接 dispatch 不发）
- **后端**：`NotificationService` 的 `notifyMatchChange` / `notifyExamResult` 等无人调用
- **后端**：SSE 和 Notification 两个系统完全断开
- **前端**：Learning 页没有任何 store，纯 useState，切页即丢
- **前端**：无事件总线，页面间无法感知对方动作

## 二、前端方案

### Phase 1：workspace 事件 Store（~80行新建）

**文件：** `frontend/src/stores/workspace.ts`

```ts
import { create } from 'zustand';

// ── 跨页面事件类型 ──
export type WorkspaceEvent =
  | { type: 'path_generated'; planId: number; planName: string; totalSkills: number }
  | { type: 'skill_completed'; skillName: string; newMatchScore?: number }
  | { type: 'exam_completed'; skillName: string; passed: boolean; score: number }
  | { type: 'match_updated'; newScore: number; jobTitle?: string }
  | { type: 'agent_task_completed'; taskId: string; agentType: string; skillName?: string }
  | { type: 'agent_task_started'; taskId: string; agentType: string }
  | { type: 'resource_ready'; skillName: string; contentType: string }
  | { type: 'today_tasks_refresh'; };

interface WorkspaceState {
  // ── 共享数据 ──
  matchScore: number;
  agentBusyCount: number;
  agentIdleCount: number;
  unreadNotificationCount: number;

  // ── 事件系统 ──
  lastEvent: WorkspaceEvent | null;
  eventSeq: number;                   // 单调递增，确保 subscribe 能检测到同类型事件

  // ── Actions ──
  emit: (event: WorkspaceEvent) => void;
  setMatchScore: (score: number) => void;
  setAgentCounts: (busy: number, idle: number) => void;
  setUnreadCount: (count: number) => void;
}

export const useWorkspaceStore = create<WorkspaceState>((set) => ({
  matchScore: 0,
  agentBusyCount: 0,
  agentIdleCount: 0,
  unreadNotificationCount: 0,
  lastEvent: null,
  eventSeq: 0,

  emit: (event) => set((s) => ({
    lastEvent: event,
    eventSeq: s.eventSeq + 1,
  })),

  setMatchScore: (score) => set({ matchScore: score }),
  setAgentCounts: (busy, idle) => set({ agentBusyCount: busy, agentIdleCount: idle }),
  setUnreadCount: (count) => set({ unreadNotificationCount: count }),
}));
```

**不改 useChatStore 和 useOfficeStore 的结构。** workspace store 只管跨页事件和全局共享数据。

### Phase 2：useWorkspaceSync Hook（~60行新建）

**文件：** `frontend/src/hooks/useWorkspaceSync.ts`

```ts
import { useEffect, useRef } from 'react';
import { useWorkspaceStore } from '../stores/workspace';
import { toast } from 'react-hot-toast';

/**
 * 在 App 根层或 UserLayout 中调用一次，监听所有跨页面事件并执行副作用。
 * 各页面只管 emit，不用关心其他页面是否在线。
 */
export function useWorkspaceSync() {
  const seqRef = useRef(0);

  useEffect(() => {
    return useWorkspaceStore.subscribe(
      (state) => state,
      (state) => {
        // 只处理新事件
        if (state.eventSeq <= seqRef.current || !state.lastEvent) return;
        seqRef.current = state.eventSeq;

        const e = state.lastEvent;
        switch (e.type) {
          case 'path_generated':
            toast.success(`📚 学习路径「${e.planName}」已生成（${e.totalSkills}个技能）`);
            break;

          case 'skill_completed':
            toast.success(`🎉 技能「${e.skillName}」学习完成！`);
            if (e.newMatchScore !== undefined) {
              toast(`📈 匹配度更新为 ${e.newMatchScore}%`, { icon: '🎯' });
            }
            break;

          case 'exam_completed':
            if (e.passed) {
              toast.success(`✅ 「${e.skillName}」考试通过！得分 ${e.score}`);
            } else {
              toast(`📝 「${e.skillName}」考试未通过（${e.score}分），加油！`, { icon: '💪' });
            }
            break;

          case 'match_updated':
            toast(`🎯 匹配度更新为 ${e.newScore}%`, {
              icon: e.newScore >= 80 ? '🚀' : '📈',
            });
            break;

          case 'agent_task_completed':
            toast.success(`🤖 Agent 任务完成：${e.agentType}`);
            break;

          case 'agent_task_started':
            toast(`⚡ Agent 开始工作：${e.agentType}`, { icon: '🔄' });
            break;

          case 'resource_ready':
            toast.success(`📄 资源已就绪：${e.skillName}（${e.contentType}）`);
            break;

          case 'today_tasks_refresh':
            // 静默事件，Learning 页面内部监听，不弹 toast
            break;
        }
      }
    );
  }, []);
}
```

**调用位置：** `App.tsx` 或 `UserLayout.tsx`，全局挂载一次。

### Phase 3：各页面接入（改造量最小化）

#### 3.1 Chat.tsx — 在 handleSend 后发射事件

**改动范围：** 仅 `handleSend` 函数内，加约 15 行

```tsx
// 在 handleSend 中，收到 res.data.actions 后：
import { useWorkspaceStore } from '../../stores/workspace';

// ... 现有代码在 res.data 处理完后，加：

const emit = useWorkspaceStore.getState().emit;
if (res.data?.actions) {
  for (const action of res.data.actions) {
    switch (action.type) {
      case 'path_generated':
        emit({
          type: 'path_generated',
          planId: action.data.planId,
          planName: action.data.planName,
          totalSkills: action.data.totalSkills,
        });
        break;
      case 'target_set':
        // 目标设置后，匹配度可能变化，请求最新值
        // （后端 match_update SSE 会推送，这里不需要额外处理）
        break;
      case 'video':
      case 'animation':
      case 'diagram':
        emit({
          type: 'resource_ready',
          skillName: action.data.skillName,
          contentType: action.type,
        });
        break;
    }
  }
}
```

#### 3.2 LearningPaths.tsx — 订阅外部事件 + 发射完成事件

**改动范围：** 加一个 useEffect + 技能完成时加 emit，约 20 行

```tsx
import { useWorkspaceStore } from '../../stores/workspace';

// ① 订阅外部事件：当 Chat 生成路径后，自动刷新 Learning 数据
useEffect(() => {
  return useWorkspaceStore.subscribe(
    (state) => state.lastEvent,
    (event) => {
      if (!event) return;
      if (event.type === 'path_generated') {
        fetchPaths();  // 重新拉取学习路径列表
      }
      if (event.type === 'today_tasks_refresh') {
        fetchPaths();  // 今日任务刷新
      }
    }
  );
}, []);

// ② 在技能完成的逻辑处（KnowledgeDetail 回来后），发射事件
//    这里有两种方式：
//    a) 在 LearningPaths 的 useEffect 中检测 URL 回来时刷新数据后对比
//    b) 在 KnowledgeDetail 页面完成时直接 emit（更好的方案）
```

#### 3.5 Learning 页添加 SSE 监听

Learning 页当前无 SSE。加一个简单的 SSE 消费，监听 `resource_ready` 和 `match_update`：

```tsx
import { useSSE } from '../../hooks/useSSE';

// 在 LearningPaths 中：
useSSE((event) => {
  if (event.type === 'match_update') {
    useWorkspaceStore.getState().setMatchScore(event.data.score);
  }
  if (event.type === 'resource_ready') {
    // 重新拉取今日任务（因为资源就绪可能影响任务状态）
    fetchPaths();
  }
});
```

### Phase 4：Office Store 补充 Agent 计数

**改动范围：** `office.ts` 加两个派生值或 computed

在 AgentPanel 现有的 SSE `graph_node` 事件处理中，每次更新 `agentStatuses` 后，顺便计算 busy/idle 计数并同步到 workspace store：

```tsx
// useGraphEvents.ts 或 AgentPanel 中
const agentStatuses = useOfficeStore((s) => s.agentStatuses);
useEffect(() => {
  const statuses = Object.values(agentStatuses);
  const busy = statuses.filter(s => s === 'working').length;
  const idle = statuses.filter(s => s === 'idle').length;
  useWorkspaceStore.getState().setAgentCounts(busy, idle);
}, [agentStatuses]);
```

## 三、后端方案

### Phase A：AgentOffice 补 SSE 发射（关键）

**问题：** `agent-office.controller.ts` 的 `executeTask()` 异步函数不发 SSE 事件。只有 queue processor 发。用户从 AgentOffice 直接派发的任务，其他页面收不到实时通知。

**改动文件：** `backend-ts/src/modules/agent-office/agent-office.controller.ts`

**改动内容：** 在 `executeTask` 方法中注入 `EventsService` 并在关键节点发射事件：

```ts
// 1. controller 构造函数注入 EventsService
constructor(
  private readonly eventsService: EventsService,
  // ... 现有依赖
) {}

// 2. executeTask 方法中加 emit：

// 任务开始执行时
this.eventsService.emitAgentStatus(userId, agentType, 'working', `开始执行: ${title}`);
this.eventsService.emitAgentProgress(userId, agentType, taskId, 30, '处理中...');

// 任务完成时
this.eventsService.emitAgentStatus(userId, agentType, 'idle');
this.eventsService.emitAgentProgress(userId, agentType, taskId, 100, '完成');
this.eventsService.emitResourceReady(userId, skillName, agentType);

// 任务失败时
this.eventsService.emitAgentStatus(userId, agentType, 'idle', 'error');
```

**约 +12 行代码。**

### Phase B：Notification → SSE 桥接

**问题：** `NotificationService` 创建通知后不发 SSE。`EventsService` 有 `emitNotification` 但没人调。两个系统断开。

**改动文件：** `backend-ts/src/services/notification.service.ts`

**改动内容：** 在 `create` 方法中自动桥接到 SSE：

```ts
// 在 create() 方法中，notification 保存后加：
if (notification.userId) {
  this.eventsService.emitNotification(notification.userId, {
    id: notification.id,
    title: notification.title,
    type: notification.type,
  });
  // 同时更新未读计数
  const count = await this.getUnreadCount(notification.userId);
  this.eventsService.emit(notification.userId, {
    type: 'unread_count',
    data: { count },
  });
}
```

**约 +8 行代码。**

│                                                                     │
│  Chat.handleSend ──emit──► workspace store ──subscribe──► toast     │
│                                    │                                │

## 五、实施顺序

```
Phase 1: workspace.ts store          ← 无依赖，立即开始
    ↓
Phase 2: useWorkspaceSync hook       ← 依赖 Phase 1
    ↓
Phase 3: 各页面接入 emit/subscribe   ← 依赖 Phase 1+2
  ├─ Chat.tsx emit                    ← 改动最小，先做
  ├─ LearningPaths subscribe          ← 验证联动效果
  ├─ AgentOffice emit                 ← 验证跨页通知
  └─ KnowledgeDetail emit             ← 验证技能完成联动
    ↓
Phase A: 后端 AgentOffice 补 SSE     ← 可与前端并行
Phase B: 后端 Notification→SSE 桥接  ← 可与前端并行
Phase C: 后端业务节点触发 Notification ← 依赖 Phase B

```

## 六、验收标准

| 场景 | 预期行为 | 涉及页面 |
|------|----------|----------|
| Chat 说"生成学习路径" | Chat 显示路径卡片 + toast + 切到 Learning 页路径已存在 | Chat → Learning |
| Learning 完成一个技能 | toast "🎉 技能完成" + TopBar 匹配度更新（如切换到 Learning 页） | Knowledge → Learning |
| AgentOffice 派发任务 | toast "⚡ Agent 开始工作" + Chat 页 AgentPanel 显示工作中 | AgentOffice → Chat |
| AgentOffice 任务完成 | toast "🤖 任务完成" + AgentPanel 状态更新 | AgentOffice → Chat |
| 考试通过 | toast "✅ 考试通过" + 匹配度更新 + 通知中心有新通知 | Exam → 全局 |
| 匹配度 >= 80% | toast "🚀 可以投递了" + 通知中心 | Match → 全局 |

## 七、不做的事

| 不做 | 理由 |
|------|------|
| 不改布局（WorkspaceLayout/Rail/TopBar） | 你说了不要三合一 |
| 不改路由结构 | 保持现有 /user/chat, /user/learning, /user/agent-office |
| 不改 useChatStore 结构 | 消息存储独立，只加 emit 调用 |
| 不改 useOfficeStore 结构 | 只在 useGraphEvents 中同步计数到 workspace store |
| 不合并 Agent 模型 | 前端 AGENT_CONFIGS vs 后端 agent_profiles_v3 的双轨制保持现状 |
| 不拆 AgentOffice.tsx | 1053 行虽然大，但不在本次范围内 |
