### 发现的问题

**[CRITICAL] 今日任务永远为空 — 时间预算算法缺陷**

- 文件：`backend-ts/src/services/task-scheduler.service.ts` 第 258 行
- 根因：`generateTodayTasks()` 的分配逻辑要求 `mainUsed + estimatedMin <= mainMinutes`
- 实际值：`mainMinutes = 96`（每天2小时 × 80%），但模板中最小技能 `HTML/CSS` 需要 150 分钟
- 结果：所有技能都超过预算，循环结束后 `tasks` 数组仍为空
- 对比：`PlannerAgentService.generateDailyTasks`（第 333 行）有 `&& tasks.length > 0` 保底逻辑，但 `TaskSchedulerService` 没有

**[MEDIUM] Dashboard.tsx：火花图使用硬编码假数据**

- 文件：`frontend/src/pages/user/Dashboard.tsx` 第 241-242 行
- 代码：`[2, 3, 1, 4, 3, 2, 5]` + 注释 `// 真实数据到位后替换`

| 生成讲义 `POST /api/user/agents/lecture` | ⚠️ | LLM 生成成功，但**未保存到知识库** |
| 标记已读 `POST /api/user/progress/read` | ❌ | 返回成功但**数据未持久化** |
| 标记完成 `POST /api/user/progress/complete` | ❌ | 返回成功但**数据未持久化** |
| 技能掌握度 | ❌ | 始终 `masteryPct=0.00`，从未更新 |
| 匹配度更新 | ❌ | 始终为 0 |

### 发现的问题

**[CRITICAL] 进度数据不持久化 — TypeORM JSON 脏检查失败**

- 文件：`backend-ts/src/modules/progress/progress.controller.ts` 第 141、175 行
- 根因：`updateSkillProgress()` 中 `const pathData = path.pathData || {}` 获取的是**同一引用**，原地修改后赋值回去，TypeORM 的变更检测认为没有变化，跳过 `path_data` 列的 UPDATE
- 影响：`read`、`quiz`、`done` 三种操作全部无效
- 修复：使用深拷贝 `JSON.parse(JSON.stringify(path.pathData || {}))`

**[CRITICAL] 进度更新不联动 user_skills_v3 表**

- 文件：`backend-ts/src/modules/progress/progress.controller.ts`（整个文件）
- 问题：ProgressController 从未注入或调用 `SkillService`，标记已读/完成只更新 `learning_plans_v3.path_data`，不更新 `user_skills_v3.mastery_pct`
- 后果：匹配度计算依赖 `user_skills_v3`，所以匹配度永远为 0

**[CRITICAL] 讲义生成不保存到知识库**

- 文件：`backend-ts/src/modules/agents/agents.controller.ts` 第 27-31 行
- 问题：`AgentsController` 直接返回 LLM 输出，未调用 `KnowledgeBaseService.saveLecture()`
- 后果：每次访问知识内容都返回 `has_content: false`，每次生成讲义都重新调用 LLM（浪费成本）

**[MEDIUM] `read` 操作不更新学习路径状态**

- 文件：`progress.controller.ts` 第 149-150 行
- 问题：`read` 操作只设置 `skill.read_at`，不改变 `skill.status`（仍为 `pending`）

**[MEDIUM] KnowledgeDetail.tsx：无内容时无生成按钮**

- 文件：`frontend/src/pages/user/KnowledgeDetail.tsx` 第 109-122 行
- 问题：`has_content: false` 时只显示"暂无内容"和返回按钮，无法触发内容生成

**[HIGH] KnowledgeDetail.tsx：API 失败时生成假内容**

- 文件：`frontend/src/pages/user/KnowledgeDetail.tsx` 第 57-67 行
- 问题：`.catch()` 中根据技能名拼接假讲义内容，与真实内容无法区分

### 发现的问题

**[CRITICAL] 匹配度永远为 0 — 技能匹配阈值过高**

- 文件：`backend-ts/src/services/match-agent.service.ts` 第 213 行
- 根因：`if (userSkill && userSkill.masteryPct >= 60)` — 只有掌握度 ≥ 60% 才算"匹配"

- 设计缺陷：混淆了"没有这个技能"和"有这个技能但掌握度低"两个概念
